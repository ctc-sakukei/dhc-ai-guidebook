## 📝 文档命名规范

```
<编号>-<文档标题>.md
```

**示例**：
- `0-基本知识.md`（入门）
- `120-高级Prompt技巧.md`（进阶）
- `500-Agent与Skill开发.md`（高级）
- `900-设计哲学.md`（杂项）

**注意事项**：
- 编号不需要补零（使用 `1` 而非 `001`）
- 标题使用中文，简洁明了
- FAQ 文档固定使用 `99`、`499`、`899` 编号

## ✍️ 新增文档流程

1. **确定分类**：根据目标读者选择合适的编号范围
2. **选择编号**：在该范围内选择未使用的编号
3. **创建文档**：按照命名规范创建 Markdown 文件
4. **遵循规范**：按照对应分类的文档规范编写内容
5. **更新索引**：在本文件中添加文档链接（如需要）

## 🔄 文档维护原则

- **保持更新**：Claude Code 更新后及时修订相关文档
- **统一术语**：使用一致的专业术语和表达方式
- **交叉引用**：在相关文档间添加链接，便于导航
- **用户反馈**：根据 FAQ 中的高频问题优化文档内容

## 📌 通用编写规范

所有文档都应遵循以下基本规范：

1. **语言风格**：统一使用"用户"而非"读者"
2. **格式规范**：严格遵循 Markdown 语法
3. **示例质量**：所有示例必须经过实际测试
4. **版本标注**：涉及版本特性时明确标注适用版本
5. **避免冗余**：不要在多个文档中重复相同内容
