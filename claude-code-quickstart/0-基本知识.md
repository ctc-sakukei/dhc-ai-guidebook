# 基本知识

> 了解以下知识可以更好地理解和使用 Claude Code。

---

## AI 核心概念

### Prompt（提示词）

给 AI 模型的输入指令或问题。好的 Prompt 应该**清晰、具体**，包含足够的上下文信息。

**示例对比：**
- ❌ 差：`"写代码"`
- ✅ 好：`"用 Python 写一个快速排序函数"`

---

### Token

AI 模型处理文本的基本单位，类似于"字块"。

- **英文**：1 Token ≈ 0.75 个单词
- **中文**：1 Token ≈ 0.5 个汉字
- **影响因素**：Token 数量直接影响**成本**和**处理速度**

---

### Temperature（温度）

控制 AI 输出**随机性**的参数，取值范围：`0 ~ 1`

| 温度值 | 输出特点 | 适用场景 |
|--------|---------|---------|
| **低温度**（0.0 - 0.3） | 确定、保守、一致性高 | 代码生成、技术文档 |
| **中温度**（0.4 - 0.7） | 平衡创意与稳定性 | 通用对话、内容创作 |
| **高温度**（0.8 - 1.0） | 创意、多样、不可预测 | 创意写作、头脑风暴 |

---

### Hallucination（幻觉）

AI 模型生成**看似合理但实际错误或虚构**的内容。

**常见表现：**
- 编造不存在的 API 或函数名
- 虚构事实、数据或引用
- 混淆相似概念

> ⚠️ **注意**：使用时需要验证关键信息的准确性，尤其是涉及技术细节时。

---

### Context Window（上下文窗口）

AI 模型一次能处理的**最大 Token 数量**。

**Claude Sonnet 4.5 的上下文窗口：**
- **容量**：200K Token
- **中文内容**：约 15 万字
- **包含内容**：对话历史 + 输入文档 + 代码库信息

## Claude Code 介绍

### Claude

**Anthropic** 公司开发的大语言模型系列，以**安全性和可靠性**著称。

**核心特性：**
- 长上下文理解（200K Token）
- 多模态输入（文本、图像）
- 擅长复杂推理和代码生成

---

### Claude Code

基于 Claude 的 **AI 编程助手**，是一个完整的 **Agent 系统**。

**核心能力：**
- 自主读写文件
- 执行命令行操作
- 智能搜索代码库
- 全流程开发（需求分析 → 代码实现 → 测试优化）

---

### Claude 模型对比

Claude 提供三个层级的模型，满足不同场景需求：

| 模型 | 定位 | 特点 | 适用场景 | 成本 |
|------|------|------|---------|------|
| **Opus 4.6** | 最强大 | 深度推理、架构设计能力 | 复杂系统设计、高难度任务 | 最高 |
| **Sonnet 4.6** | 平衡性能与成本 | 响应快、能力强（**默认模型**） | 日常开发、通用编程任务 | 中等 |
| **Haiku 4.5** | 快速轻量 | 延迟低、响应迅速 | 代码补全、简单重复任务 | 最低 |

**选择建议：**
- 需要深度思考和精确输出 → **Opus 4.6**
- 日常开发任务 → **Sonnet 4.6**（推荐）
- 快速补全、简单任务 → **Haiku 4.5**

市场上主流的 AI 编程工具对比：

| 工具 | 核心能力 | 特点 |
|------|---------|------|
| **GitHub Copilot** | 代码补全 | 与 GitHub 深度集成，擅长行内补全 |
| **Cursor** | Agent 能力 | 类似 Claude Code，强调上下文理解 |
| **Windsurf** | Agent 能力 | 提供可视化 Agent 流程 |
| **Claude Code** | Agent 能力 | 强大的推理能力和长上下文支持 |
