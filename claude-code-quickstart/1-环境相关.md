## CTC 环境说明

### 使用方式

开发人员**在 WSL 环境中使用 Claude Code**，通过 **AWS Bedrock 账号**登录。

**背景说明：**

由于 Claude 的地域限制，国内无法直接访问 Claude API，且每个账号都需要绑定手机号。CTC 采用 AWS Bedrock 提供的 Claude 模型服务。

**环境架构：**

```
Windows 开发机
    ↓
WSL (Linux 子系统)
    ↓
Claude Code CLI (使用 AWS Bedrock 账号登录)
    ↓
AWS Bedrock API
    ↓
Claude 模型
```

**使用 WSL 的原因：**

- **AWS CLI 兼容性**：AWS 认证和配置工具在 Linux 环境下更稳定
- **Claude Code 性能**：Linux 环境下运行性能更好
- **开发工具生态**：更丰富的命令行工具和包管理器

## 使用方式

### 推荐方式：VSCode Remote - WSL ⭐⭐⭐⭐⭐

这是 **CTC 项目的标准开发方式**，体验最佳。

#### 配置步骤

1. **安装 VSCode 扩展**

   在 Windows 的 VSCode 中安装 `Remote - WSL` 扩展

2. **连接到 WSL**

   - 点击 VSCode 左下角绿色按钮
   - 选择 `Connect to WSL`
   - 或使用快捷键 `F1` → 输入 `WSL: Connect to WSL`

3. **打开项目**

   - 在 WSL 环境中打开项目文件夹
   - ✅ **推荐路径**：`~/projects/your-project`（WSL 本地文件系统）
   - ❌ **避免路径**：`/mnt/c/projects/`（Windows 挂载目录，性能差）

4. **安装 Claude Code 扩展**

   在 WSL 环境中安装 `Claude Code` 扩展（会提示 "Install in WSL"）

5. **开始使用**

   - 点击侧边栏 Claude Code 图标
   - 或按 `Ctrl+Shift+P` → 输入 `Claude Code`

#### 优势

- ✅ **无缝集成**：IDE、终端、Claude Code 都在 WSL 环境中
- ✅ **性能最佳**：避免跨文件系统访问
- ✅ **实时同步**：AI 修改文件后立即在编辑器中显示
- ✅ **完整功能**：支持所有 Claude Code 特性

#### 架构示意图

```
┌─────────────────────────────────────┐
│       Windows (你的操作界面)        │
│  ┌───────────────────────────────┐  │
│  │      VSCode (编辑器界面)      │  │
│  │         ↓ Remote-WSL          │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
              ↓ 远程连接
┌─────────────────────────────────────┐

│  │   Claude Code CLI             │  │
│  │   AWS CLI (连接 Bedrock)      │  │
```

### 备用方式：命令行 + 直接文件访问 ⭐⭐⭐

适合临时使用或快速验证环境。

#### 使用步骤

1. **Windows 中打开项目**

   用 VSCode 打开：`C:\projects\my-project`

2. **WSL 中运行 Claude Code**

   ```bash
   cd /mnt/c/projects/my-project
   claude-code
   ```

3. **查看修改**

   AI 修改文件后，VSCode 会自动检测并刷新

#### 注意事项

- ⚠️ **性能问题**：跨文件系统访问速度慢（约 10 倍性能差距）
- ⚠️ **路径对应关系**：
  - Windows: `C:\Users\username\projects`
  - WSL: `/mnt/c/Users/username/projects`

## 性能优化建议

### 项目存放位置对比

| 位置 | 路径示例 | 性能 | 推荐度 |
|------|---------|------|--------|
| **WSL 文件系统** | `~/projects/` | ⚡ 快速（基准性能） | ✅ **强烈推荐** |
| Windows 挂载目录 | `/mnt/c/projects/` | 🐌 缓慢（约 10 倍差距） | ❌ **避免使用** |

### 性能差异原因

- **WSL 本地文件系统**（`~/`）：原生速度，无转换开销
- **Windows 挂载目录**（`/mnt/c/`）：需要跨系统文件转换，性能损失严重

### 项目迁移方法

如果项目当前在 Windows 目录，建议迁移到 WSL 文件系统：

```bash
# 1. 复制项目到 WSL（保留 Git 历史）
cp -r /mnt/c/projects/my-project ~/projects/

# 2. 进入新目录
cd ~/projects/my-project

# 3. 验证 Git 状态
git status

# 4. 验证远程仓库配置（可选）
git remote -v
```
