# 高级Prompt技巧

> **难度级别**：⭐⭐ 进阶
> **目标用户**：熟悉基础使用，希望提高Prompt质量的用户

## 链式Prompt设计

将复杂任务分解为多个步骤，逐步引导Claude完成：

```
步骤1：分析现有代码结构
步骤2：识别重构机会
步骤3：提出具体改进方案
步骤4：实施改进
```

**优势**：更精确的控制，减少错误，便于中途调整。

---

## Few-shot Learning示例

提供2-3个示例，让Claude理解预期格式：

```
需求：将函数转换为箭头函数

示例1：
输入：function add(a, b) { return a + b; }
输出：const add = (a, b) => a + b;

示例2：
输入：function greet(name) { return "Hello " + name; }
输出：const greet = (name) => `Hello ${name}`;

现在处理：function multiply(x, y) { return x * y; }
```

## 角色扮演技巧

让Claude扮演特定角色以获得更专业的回答：

```
"作为资深Java架构师，请审查这个Spring Boot项目的设计..."
"以代码审查员的角度，找出这段代码的潜在问题..."
"假设你是性能优化专家，建议如何改进这个算法..."
```

## 复杂任务分解策略

### 策略1：自顶向下
```
1. 设计整体架构
2. 定义模块接口
3. 实现核心模块
4. 实现辅助模块
5. 集成测试
```

### 策略2：迭代开发
```
迭代1：实现最小可用版本
迭代2：添加核心功能
迭代3：优化和完善
```
