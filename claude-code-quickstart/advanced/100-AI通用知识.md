# AI通用知识

通用的AI知识，了解这些概念有助于理解AI应用的工作原理，在定制开发或遇到问题时会更容易理解背后的机制。

## System Prompt / User Prompt

**概念说明**：
System Prompt设定AI的角色和行为规则（如"你是专业的代码助手"），User Prompt是用户的具体问题或指令。两者配合引导AI输出。

**在Claude Code中的应用**：
- **System Prompt**：Claude Code内置的系统提示词定义了其作为代码助手的行为模式
- **User Prompt**：开发者在对话中输入的每一条指令
- **CLAUDE.md**：项目级的System Prompt扩展，增加项目特定的规则和上下文
- **Memory系统**：持久化的上下文信息，作为System Prompt的补充

**实战示例**：
```
# System Prompt（内置）
你是Claude Code，专业的代码助手，擅长代码生成、调试和重构...

# CLAUDE.md（项目级System Prompt）
项目使用Spring Boot框架，遵循RESTful API设计规范...

# User Prompt（开发者输入）
请创建一个用户管理API，包含CRUD操作

# 最终效果
Claude Code会结合系统规则+项目规范+用户指令生成代码
```

## Agent

**概念说明**：
能够自主规划和执行多步骤任务的AI系统。Agent可以调用工具、读写文件、运行代码，直到完成目标。Claude Code就是一个典型的Agent。

**在Claude Code中的应用**：
- **主对话**：Claude Code本身是一个Agent，可以规划和执行复杂的开发任务
- **Sub-Agent**：Explore、Plan等专门化的子Agent，独立上下文处理特定任务
- **自主决策**：Agent可以自动选择使用哪些工具（Read、Edit、Bash等）

**实战示例**：
```
你：请实现用户注册功能，包含邮箱验证和密码加密

Claude Code（Agent思考过程）：
1. [规划] 需要创建DTO、Service、Controller
2. [调用Read] 读取现有User实体了解结构
3. [调用Read] 读取BaseService了解规范
4. [调用Write] 生成UserRegistrationDTO
5. [调用Edit] 在UserService中添加register方法
6. [调用Edit] 在UserController中添加端点
7. [调用Bash] 运行测试验证功能

结果：自主完成多步骤任务，无需每步人工指导
```

## Copilot

**概念说明**：
辅助型AI助手，根据当前上下文提供实时建议和补全。主要用于代码编写，如GitHub Copilot在编码时自动补全代码片段。

**与Claude Code的对比**：
- **Copilot**：在IDE中实时补全，输入即建议，轻量快速
- **Claude Code**：完成完整任务，需要明确指令，功能强大

**实战场景**：
```
# Copilot使用场景
def calculate_total(items):  # 输入函数签名
    # Copilot自动补全：
    return sum(item.price * item.quantity for item in items)

# Claude Code使用场景
你：请实现订单总价计算，包括折扣、税费、运费
Claude Code：[生成完整的计算逻辑，包括多种情况处理]
```

## Agent VS Copilot

**核心区别**：
- **Agent**：主动完成完整任务（如"实现登录功能"）
- **Copilot**：被动提供建议（如自动补全代码）
- **Agent**：更自主，需要较长时间
- **Copilot**：更轻量快速，实时响应

**选择建议**：
- 编写代码时：Copilot（实时补全，流畅体验）
- 新功能开发：Agent（完整规划和实现）
- 代码重构：Agent（理解全局，系统性修改）
- 简单修改：Copilot（快速调整）
- **最佳实践**：Copilot写代码 + Agent做架构和重构

## Tool Calling / Function Calling

**概念说明**：
AI模型调用外部工具或函数的能力。例如Claude Code可以调用文件读写、命令执行等工具，从而完成代码修改、运行测试等实际操作。

**在Claude Code中的应用**：

Claude Code内置多种工具：

| 工具类别 | 工具名称 | 用途 |
|---------|---------|------|
| 文件操作 | Read | 读取文件内容 |
| | Write | 创建新文件 |
| | Edit | 修改现有文件 |
| | Glob | 查找文件 |
| 代码搜索 | Grep | 搜索代码内容 |
| 命令执行 | Bash | 运行Shell命令 |
| 项目管理 | TodoWrite | 管理任务清单 |

**实战流程**：
```
你：请修复UserService中的Bug

Claude Code内部流程：
1. [调用Grep] 搜索UserService.java位置
2. [调用Read] 读取UserService.java内容
3. [分析] 发现空指针异常问题
4. [调用Edit] 修改代码添加空值检查
5. [调用Bash] 运行测试验证修复

开发者视角：只看到最终结果，工具调用是自动的
```

**Tool Calling优势**：
- 突破纯文本限制，可以实际操作文件和系统
- 自动选择合适的工具完成任务
- 多个工具可以组合使用解决复杂问题

## Streaming (API是否以Stream的形式返回)
流式返回指AI逐字输出内容，用户能实时看到生成过程，体验更流畅。非流式则等待全部内容生成完毕后一次性返回。

## Latency (延迟)
从发送请求到收到AI响应的时间。影响因素包括模型大小、网络速度、Token数量。Streaming模式可以降低感知延迟，提升用户体验。

## MCP（Model Context Protocol）

**概念说明**：
Model Context Protocol，AI模型与外部数据源连接的标准协议。通过MCP，AI可以访问数据库、API、文件系统等资源，扩展能力边界。

**在Claude Code中的应用**：

Claude Code支持MCP服务器，可以连接各种外部资源：

### MCP实战案例1：连接数据库

**场景**：需要Claude Code直接查询生产数据库分析问题

**配置MCP Server**：
```json
// .claude/mcp-config.json
{
  "mcpServers": {
    "mysql": {
      "command": "mcp-server-mysql",
      "args": ["--host", "localhost", "--database", "myapp"]
    }
  }
}
```

**使用示例**：
```
你：查询最近一周注册用户数量的变化趋势

Claude Code（通过MCP）：
1. [调用MySQL MCP] 执行查询：
   SELECT DATE(created_at), COUNT(*)
   FROM users
   WHERE created_at >= DATE_SUB(NOW(), INTERVAL 7 DAY)
   GROUP BY DATE(created_at)
2. [分析数据] 识别异常波动
3. [生成报告] 输出趋势分析和建议

结果：直接基于真实数据分析，无需手动导出
```

### MCP实战案例2：连接内部API

**场景**：需要访问公司内部的API文档服务

**配置MCP Server**：
```json
{
  "mcpServers": {
    "internal-api": {
      "command": "mcp-server-http",
      "args": ["--base-url", "https://api-docs.company.internal"]
    }
  }
}
```

**使用示例**：
```
你：支付服务的退款API参数是什么？

Claude Code（通过MCP）：
1. [调用HTTP MCP] 查询内部API文档
2. [解析响应] 提取退款API规格
3. [输出] 清晰的参数说明和示例

优势：无需离开Claude Code查阅文档
```

### MCP实战案例3：连接Jira/项目管理

**场景**：将代码修改与Jira任务关联

**配置MCP Server**：
```json
{
  "mcpServers": {
    "jira": {
      "command": "mcp-server-jira",
      "args": ["--domain", "company.atlassian.net"]
    }
  }
}
```

**使用示例**：
```
你：实现JIRA-1234任务描述的功能

Claude Code（通过MCP）：
1. [调用Jira MCP] 获取JIRA-1234详情
2. [解析需求] 理解任务要求
3. [生成代码] 实现功能
4. [更新Jira] 自动添加评论记录进度

结果：代码和任务管理无缝集成
```

**MCP的价值**：
- 打破信息孤岛，统一访问各种数据源
- 实时数据分析，基于最新信息决策
- 自动化工作流，减少手动切换工具

## RAG（Retrieval-Augmented Generation）

**概念说明**：
Retrieval-Augmented Generation，检索增强生成。先从知识库检索相关信息，再结合这些信息生成回答，可有效减少幻觉，提供更准确的答案。

**在Claude Code中的应用**：

Claude Code的Memory系统本质上是一个轻量级RAG：

### RAG实战案例1：项目知识库

**场景**：大型项目有100+个模块，无法把所有文档都放入上下文

**RAG方案**：
```
1. 建立项目知识库
.claude/knowledge/
├── architecture/
│   ├── overview.md
│   ├── user-service.md
│   ├── order-service.md
│   └── payment-service.md
├── coding-standards/
│   ├── java-style.md
│   ├── api-design.md
│   └── error-handling.md
└── troubleshooting/
    ├── common-errors.md
    └── performance-issues.md

2. 使用RAG增强Memory
在MEMORY.md中添加索引：
```markdown
# 项目知识库索引

## 架构文档
详见：.claude/knowledge/architecture/

## 编码规范
详见：.claude/knowledge/coding-standards/

## 常见问题
详见：.claude/knowledge/troubleshooting/
```

3. Claude Code工作流程：
你：用户服务的API设计规范是什么？
Claude Code：
  → [检索] 在MEMORY.md发现architecture索引
  → [读取] .claude/knowledge/architecture/user-service.md
  → [生成] 基于检索到的规范回答问题
```

**效果**：
- 不受200行MEMORY.md限制
- 按需加载相关知识
- 减少上下文Token消耗

### RAG实战案例2：历史问题库

**场景**：团队遇到的问题和解决方案需要复用

**RAG方案**：
```markdown
# .claude/knowledge/troubleshooting/database.md

## Q: 用户查询慢（2024-03）
**症状**：用户列表加载超过3秒
**原因**：N+1查询问题
**解决方案**：
```java
// 错误
users.forEach(u -> u.setProfile(getProfile(u.id)));

// 正确
users = userRepo.findAllWithProfiles();
```

## Q: 连接池耗尽（2024-02）
**症状**：高峰期数据库连接超时
**原因**：连接未正确释放
**解决方案**：使用try-with-resources
```

**使用示例**：
```
你：数据库查询很慢，怎么优化？

Claude Code（RAG流程）：
1. [检索] 在troubleshooting/database.md找到相似问题
2. [阅读] 历史解决方案
3. [生成] 结合当前代码给出建议

优势：自动复用团队经验，不会重复踩坑
```

### RAG实战案例3：API文档检索

**场景**：使用第三方库但不记得具体API

**RAG方案**：
```
1. 将库的文档转换为向量索引
2. 使用Embedding实现语义搜索
3. 找到最相关的API说明

示例：
你：Spring Boot如何配置数据库连接池大小？

RAG系统：
1. [Embedding] 将问题转为向量
2. [检索] 从Spring Boot文档中找到相似内容
3. [提取] 相关配置说明
4. [生成] 完整的配置示例和解释

传统搜索 vs RAG：
- 关键词搜索："配置" "连接池"（可能找不到）
- RAG语义搜索：理解"数据库连接池大小"的含义（准确匹配）
```

### Embedding

**概念说明**：
将文本转换为数字向量的技术，语义相似的文本向量距离更近。RAG系统用Embedding来快速找到与问题最相关的知识片段。

**工作原理**：
```
原始文本：
- "如何连接数据库"
- "数据库连接方法"
- "今天天气真好"

转换为向量：
- [0.2, 0.8, 0.1, ...] (数据库连接)
- [0.3, 0.7, 0.2, ...] (数据库连接)
- [0.9, 0.1, 0.8, ...] (天气)

计算相似度：
前两个向量距离近（语义相似）
第三个向量距离远（语义不同）
```

**在RAG中的应用**：
```
用户问题："数据库慢怎么办"
↓ Embedding
问题向量：[0.25, 0.75, 0.15, ...]
↓ 相似度搜索
知识库中最相似的3个文档：
1. "数据库性能优化指南" (相似度0.92)
2. "SQL查询调优方法" (相似度0.87)
3. "索引设计最佳实践" (相似度0.81)
↓ 提供给Claude Code
基于这些文档生成回答
```

**实战建议**：
- 小项目：使用Memory + 文件索引（简单）
- 中型项目：构建本地向量数据库（如ChromaDB）
- 大型项目：专业RAG系统（如LangChain + Pinecone）

## 如何选择合适的Claude模型

在实际使用中，需要根据项目规模和具体需求选择合适的模型：

### 大参数模型（如Opus、Sonnet）

**优点：**
- 对庞大的项目支持更好，能够理解复杂的代码库结构
- 可以处理复杂的业务逻辑和架构设计
- 支持更多的开发语言和技术栈
- 推理能力更强，能够处理更抽象的问题

**缺点：**
- 响应速度较慢，生成内容需要更多时间
- 使用成本较高（按Token计费）
- 本地部署难度大或无法本地部署（需要大量计算资源）

### 小参数模型（如Haiku及其他轻量级模型）

**优点：**
- 响应速度快，适合需要快速反馈的场景
- 使用成本低，适合大量简单任务
- 可以本地部署，对显存要求相对较低
- 适合标准化、模式化的任务

**缺点：**
- 处理复杂业务逻辑时容易出错或遗漏细节
- 主要支持主流的开发语言和技术，对小众技术栈支持有限
- 对大型项目的整体把握能力较弱
- 处理抽象概念和复杂推理时表现不足

### 选择建议

- **企业级大型项目**：推荐使用Opus或Sonnet，确保代码质量和理解深度
- **日常开发任务**：Sonnet提供了最佳的性能与成本平衡
- **简单重复任务**：可以使用Haiku或其他轻量级模型，如代码格式化、简单重构等
- **成本敏感场景**：优先使用Haiku处理简单任务，仅在必要时切换到更强大的模型
- **本地部署需求**：考虑使用开源的小参数模型，但需要接受其性能限制
