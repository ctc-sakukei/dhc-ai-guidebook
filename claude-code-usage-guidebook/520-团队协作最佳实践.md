# 520-团队协作最佳实践

> **难度级别**：⭐⭐⭐ 高级
> **目标读者**：Team Leader，负责团队Claude Code使用管理

## 团队Memory共享策略

### 共享Memory结构
```
.claude/
├── memory/
│   ├── MEMORY.md          # 项目通用记忆
│   ├── architecture.md    # 架构设计
│   ├── conventions.md     # 团队约定
│   └── troubleshooting.md # 常见问题解决
```

### Memory维护原则
- **个人Memory**：个人偏好、学习笔记
- **项目Memory**：架构决策、通用规范
- **定期同步**：每周Review，删除过时内容

## 统一Skill库管理

### Skill分类
```
.claude/skills/
├── build/         # 构建相关
├── test/          # 测试相关
├── deploy/        # 部署相关
└── code-review/   # 代码审查
```

### Skill开发流程
1. 提出需求（Issues）
2. 设计Skill（讨论）
3. 实现并测试
4. 文档化
5. 团队培训
6. 正式使用

---

## Code Review流程集成

### 审查前
```markdown
使用 /pre-review skill：
- 运行linter
- 执行测试
- 检查代码规范
```

### 审查中
```markdown
Claude辅助Review：
"请分析这个PR的变更，重点关注：
1. 是否符合项目架构
2. 是否有潜在bug
3. 是否需要补充测试"
```

### 审查后
```markdown
使用 /post-review skill：
- 生成Review总结
- 更新文档（如需要）
```

## 成本管理和预算分配

### 成本监控
```markdown
团队级别：
- 月度总成本
- 人均成本
- 项目成本

个人级别：
- 每周Token消耗
- 任务Token平均值
```

### 成本优化建议
1. **优先使用Sonnet**：性价比最高
2. **避免重复查询**：利用Memory
3. **批量处理**：减少交互次数
4. **定期Review**：识别高消耗点

## 权限和安全管理

### API Key管理
- ❌ 不要：将Key提交到代码仓库
- ✅ 推荐：使用环境变量
- ✅ 推荐：团队共享Key（小团队）或个人Key（大团队）
