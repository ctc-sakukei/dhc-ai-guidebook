# 常见问题FAQ

## Claude Code生成的代码有bug怎么办？

**A:**
1. 不要直接删除，先让Claude Code自己review找问题
2. 明确指出bug的具体表现
3. 如果Claude Code多次修复失败，考虑人工修复或换思路
4. 将常见bug记录到Memory，避免重复

```
你：这段代码在处理null值时会抛出NullPointerException，
    具体在第15行的obj.toString()。请修复。
```

## 如何让Claude Code遵守项目代码规范？

**A:**
1. 在CLAUDE.md中明确记录代码规范
2. 在Prompt中重申规范
3. 提供示例代码作为参考
4. 使用linter工具自动检查

```
你：请按照项目规范生成代码：
- 使用4空格缩进
- 变量使用驼峰命名
- 所有public方法必须有JavaDoc
- 参考现有的UserService.java风格
```

## 上下文满了但任务还没完成怎么办？

**A:**
1. 使用`/compact`压缩上下文
2. 使用缓存策略（见[核心概念.md](核心概念.md#上下文管理策略)）
3. 将任务拆分为更小的子任务
4. 使用Sub-Agent处理大型任务

```
你：请将测试结果保存到./temp/test-results.md
你：/clear
你：请读取./temp/test-results.md生成总结报告
```

## 出现"This session is being continued from a previous conversation..."提示是什么意思？

**A:**

这**不是错误**，而是上下文压缩的正常提示，表示：
- 对话上下文已达到容量限制
- 早期对话被总结成摘要以腾出空间
- 对话可以继续，但详细信息已丢失

**后果：**
- 早期的代码片段、详细讨论等无法准确回忆
- 如果后续任务依赖早期细节，可能产生偏差

**应对方法：**
1. **评估影响**：判断丢失的信息是否关键
2. **重新说明**：将重要需求、规范重新告知Claude Code
3. **依赖文件**：使用文件保存关键信息，而不是依赖对话历史
4. **预防为主**：
   - 及时使用`/clear`或`/compact`
   - 大任务分解为小任务
   - 关键中间结果保存到文件

详见[使用技巧.md](使用技巧.md#注意上下文的剩余空间)

## Claude Code一直在做错误的事情怎么纠正？

**A:**
1. 立即停止：使用`/clear`清空上下文
2. 重新开始，使用更明确的指令
3. 提供正确的示例
4. 分解任务，逐步验证

```
你：/clear
你：我重新说明需求。我需要的是...（详细说明）
    请先告诉我你的理解，不要直接动手
```

## 如何让Claude Code理解复杂业务逻辑？

**A:**
1. 分层次说明：先整体，再细节
2. 使用流程图或伪代码
3. 提供实际案例
4. 参考现有代码

```
你：订单流程如下：
1. 用户提交订单
2. 系统检查库存
   - 如果库存充足：锁定库存，进入步骤3
   - 如果库存不足：提示用户，流程结束
3. 调用支付接口
   - 如果支付成功：扣减库存，创建订单，进入步骤4
   - 如果支付失败：释放库存，提示用户，流程结束
4. 发送订单确认邮件

请基于这个流程实现OrderService.createOrder方法
```

## Claude Code生成的测试用例不够全面怎么办？

**A:**
1. 明确列出需要覆盖的场景
2. 让Claude Code分析可能的边界条件
3. 参考现有的测试风格

```
你：请为login方法生成测试，需要覆盖：
1. 正常登录（用户名密码正确）
2. 用户名不存在
3. 密码错误
4. 用户被锁定
5. 参数为null
6. 参数为空字符串
参考UserServiceTest.java的测试风格
```

## 如何提高Claude Code的响应速度？

**A:**
1. 减少上下文长度（及时清理）
2. 明确指定文件范围
3. 避免让Claude Code读取大文件
4. 使用`/fast`模式（牺牲质量换速度）
5. 使用Haiku模型处理简单任务

## Claude Code修改了不该改的文件怎么办？

**A:**
1. 使用git回退：`git checkout -- file.java`
2. 在IDE中使用撤销功能
3. 下次明确说明只修改哪些文件

```
你：请只修改UserService.java文件，不要动其他文件
```

## Claude Code能完全替代人工开发吗？

**A:**

**不能。** Claude Code是强大的辅助工具，但无法替代人类开发者：

- ✅ Claude Code擅长：生成样板代码、编写测试、重构、文档生成
- ❌ Claude Code不擅长：复杂架构设计、业务决策、创新性解决方案
- ⚠️ 必须人工：代码review、安全审查、性能优化、生产部署

**正确定位**：将Claude Code视为高效的助手，而不是完全的替代品。
