# AI通用知识

了解这些通用的AI概念，有助于更好地使用各类AI工具。本文档聚焦于日常使用AI工具时需要理解的核心概念，帮助你明白工具的工作方式和适用场景。

## System Prompt / User Prompt

**概念说明**：
System Prompt 设定 AI 的角色和行为规则（如"你是专业的代码助手"），User Prompt 是用户的具体问题或指令。两者配合引导 AI 输出。

**为什么重要**：
- **System Prompt** 就像给 AI 发一份"工作说明书"，定义它的身份、能力边界和行为准则
- **User Prompt** 是实际的工作指令，告诉 AI 当前要做什么
- 理解这个概念有助于编写更有效的项目配置文件（如 CLAUDE.md）和日常指令

**实战示例**：
```
# System Prompt（内置规则）
你是专业的代码助手，擅长代码生成、调试和重构...

# 项目级 System Prompt（通过配置文件）
项目使用 Spring Boot 框架，遵循 RESTful API 设计规范...

# User Prompt（用户输入）
请创建一个用户管理 API，包含 CRUD 操作

# 最终效果
AI 会结合系统规则 + 项目规范 + 用户指令生成代码
```

**最佳实践**：
- System Prompt 应该简洁明确，避免过度约束
- User Prompt 要具体清晰，说明预期结果
- 项目级配置（如 CLAUDE.md）本质上是扩展 System Prompt

## Agent

**概念说明**：
能够自主规划和执行多步骤任务的 AI 系统。Agent 可以调用工具、读写文件、运行代码，直到完成目标。

**为什么重要**：
- 传统 AI 只能回答问题，Agent 能完成任务
- 理解 Agent 的工作方式，能更好地分配任务和设定预期
- 知道什么时候需要人工介入，什么时候可以让 Agent 自主完成

**实战示例**：
```
你：请实现用户注册功能，包含邮箱验证和密码加密

Agent 自主执行：
1. [规划] 分析需要创建 DTO、Service、Controller
2. [调用工具] 读取现有代码了解项目结构
3. [调用工具] 创建或修改必要的文件
4. [调用工具] 运行测试验证功能
5. [反馈] 报告完成情况

结果：自主完成多步骤任务，无需每步人工指导
```

**与传统 AI 的区别**：
- 传统 AI：回答问题、生成文本
- Agent：完成任务、操作系统、自主决策

## Copilot

**概念说明**：
辅助型 AI 助手，根据当前上下文提供实时建议和补全。主要用于编写代码时的自动补全和建议。

**为什么重要**：
- Copilot 追求速度和流畅度，适合编码过程中的实时辅助
- 理解 Copilot 的特点，能更好地与 Agent 类工具配合使用
- 知道何时需要"快速建议"，何时需要"完整实现"

**实战场景**：
```
# Copilot 使用场景（实时补全）
def calculate_total(items):  # 输入函数签名
    # Copilot 自动建议：
    return sum(item.price * item.quantity for item in items)

# 特点
✅ 响应快速（毫秒级）
✅ 上下文感知（基于当前文件）
❌ 只提供建议，不执行操作
```

## Agent VS Copilot

**核心区别**：

| 特性 | Agent | Copilot |
|------|-------|---------|
| **工作方式** | 主动完成完整任务 | 被动提供建议 |
| **响应速度** | 较慢（秒级） | 极快（毫秒级） |
| **任务复杂度** | 多步骤、跨文件 | 单行或代码块 |
| **执行能力** | 可以操作文件、运行命令 | 只提供文本建议 |
| **适用场景** | 功能开发、重构、调试 | 编码时的实时补全 |

**选择建议**：
- **编写代码时**：Copilot（实时补全，流畅体验）
- **新功能开发**：Agent（完整规划和实现）
- **代码重构**：Agent（理解全局，系统性修改）
- **简单修改**：Copilot（快速调整）
- **最佳实践**：Copilot 写代码 + Agent 做架构和重构

**实战对比**：
```
任务：实现订单总价计算，包括折扣、税费、运费

Copilot 的处理方式：
你开始写代码，Copilot 逐行建议补全
→ 需要你决定整体结构
→ 快速但需要人工整合

Agent 的处理方式：
你描述需求，Agent 自动实现
→ 考虑边界情况和错误处理
→ 生成完整的解决方案
```
