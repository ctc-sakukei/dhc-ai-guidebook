# Claude Code 进阶指南

本文介绍成本控制、安全注意事项和常见问题解决方案。

## 成本控制策略

由于Claude服务按Token计费，合理控制成本对长期使用至关重要。

### Token消耗来源

1. **上下文（输入）**：每次对话包含历史记录
2. **生成（输出）**：Claude Code生成的所有内容
3. **文件读取**：每次读取文件消耗Token
4. **重复工作**：因错误导致的返工

### 成本优化策略

#### 策略1：及时清理上下文

```bash
# 完成任务后立即清空
/clear

# 定期压缩
/compact
```

**节省效果**：可减少50%以上的上下文Token消耗

#### 策略2：精确指定文件范围

```
❌ 成本高：请阅读整个src目录

✅ 成本低：请阅读src/services/UserService.java
```

**节省效果**：避免不必要的文件读取，节省30-50% Token

#### 策略3：使用缓存策略

对于大型任务，使用文件缓存而不是依赖上下文：

```
你：请测试case1-3，结果保存到./temp/result1.md
你：/clear
你：请测试case4-6，结果保存到./temp/result2.md
你：/clear
你：请读取所有result文件生成总结报告
```

**节省效果**：对于10个以上测试用例的场景，可节省70% Token

#### 策略4：选择合适的模型

| 任务类型 | 推荐模型 | 成本 |
|---------|---------|------|
| 简单代码生成 | Haiku | 低 |
| 日常开发 | Sonnet | 中 |
| 复杂架构设计 | Opus | 高 |

**节省效果**：合理使用Haiku可节省60-80%成本

#### 策略5：一次性说清楚

```
❌ 多次对话（成本高）：
你：生成一个用户类
Claude Code：[生成基础类]
你：加上验证
Claude Code：[添加验证]
你：再加上toString
Claude Code：[添加toString]

✅ 一次说清（成本低）：
你：生成用户类，包含id、name、email字段，
    添加javax.validation验证，重写toString
Claude Code：[一次生成完整代码]
```

**节省效果**：减少70%的对话轮次

### 具体成本计算案例

#### 案例1：创建REST API（小型任务）

**任务描述**：创建一个用户管理API，包含CRUD操作

**Token消耗明细**：

| 步骤 | 操作 | Token消耗 | 说明 |
|-----|------|----------|------|
| 1 | 说明需求 | 200 | 输入Prompt |
| 2 | 读取现有代码 | 1,500 | 读取BaseController、BaseService等参考文件 |
| 3 | 生成Controller | 1,800 | 输出UserController代码 |
| 4 | 生成Service | 1,200 | 输出UserService代码 |
| 5 | 生成DTO类 | 600 | 输出请求/响应DTO |
| 6 | 生成测试代码 | 2,000 | 输出单元测试 |
| **总计** | - | **7,300** | 约0.22美元（按Sonnet价格） |

**成本优化建议**：
- 如果不需要测试代码，可节省2,000 Token
- 使用Haiku模型可降低成本至0.04美元

#### 案例2：代码审查（中型任务）

**任务描述**：审查一个包含5个文件的功能模块

**Token消耗明细**：

| 步骤 | 操作 | Token消耗 | 说明 |
|-----|------|----------|------|
| 1 | 说明审查要求 | 300 | 输入Prompt（安全、性能、规范） |
| 2 | 读取5个文件 | 8,000 | 平均每文件1,600 Token |
| 3 | 输出审查报告 | 3,500 | 详细问题列表和建议 |
| 4 | 修改代码 | 2,500 | 根据审查结果修改 |
| **总计** | - | **14,300** | 约0.43美元（按Sonnet价格） |

**成本陷阱**：
- ❌ 让Claude Code自动扫描目录：可能读取大量无关文件
- ✅ 明确指定需要审查的文件列表

#### 案例3：架构设计与实现（大型任务）

**任务描述**：设计并实现一个支付模块，包含设计文档、核心代码、测试

**Token消耗明细**：

| 步骤 | 操作 | Token消耗 | 说明 |
|-----|------|----------|------|
| 1 | 需求讨论 | 1,000 | 多轮对话明确需求 |
| 2 | 架构设计 | 3,000 | 输出架构文档 |
| 3 | 读取项目代码 | 15,000 | 了解现有架构和规范 |
| 4 | 生成核心代码 | 12,000 | 分批生成（避免超过4096限制） |
| 5 | 生成配置文件 | 1,500 | 数据库脚本、配置等 |
| 6 | 生成测试代码 | 8,000 | 单元测试+集成测试 |
| 7 | 代码调整 | 5,000 | 根据反馈修改 |
| 8 | 文档完善 | 2,500 | API文档、使用说明 |
| **总计** | - | **48,000** | 约1.44美元（按Sonnet价格） |

**优化策略**：
- 使用`/clear`在不同阶段间清理上下文，可节省20% Token
- 设计阶段使用Sonnet，代码生成使用Haiku，可节省40%成本
- 将中间结果保存到文件，避免重复读取，可节省30% Token

#### 案例4：大规模重构（特大型任务）

**任务描述**：将旧框架代码迁移到新框架，涉及20个文件

**Token消耗明细（未优化）**：

| 步骤 | 操作 | Token消耗 |
|-----|------|----------|
| 1 | 分析旧代码 | 40,000 |
| 2 | 设计迁移方案 | 5,000 |
| 3 | 逐文件迁移 | 80,000 |
| 4 | 调试修复 | 30,000 |
| **总计** | - | **155,000** |

**成本**：约4.65美元（按Sonnet价格）

**优化后Token消耗**：

| 步骤 | 操作 | Token消耗 | 优化方法 |
|-----|------|----------|---------|
| 1 | 分析旧代码 | 15,000 | 使用Agent独立分析，只返回摘要 |
| 2 | 设计方案 | 5,000 | 不变 |
| 3 | 迁移（分批） | 45,000 | 每迁移5个文件就/clear |
| 4 | 调试 | 12,000 | 使用缓存策略保存中间结果 |
| **总计** | - | **77,000** | 节省50% |

**成本**：约2.31美元（按Sonnet价格）

#### 实际场景成本对比

**场景1：生成一个简单的CRUD Controller（约2000 Token输出）**

| 模型 | 输入Token | 输出Token | 总成本 | 相对成本 |
|-----|---------|---------|--------|---------|
| Haiku | 1,000 | 2,000 | $0.011 | 1x |
| Sonnet | 1,000 | 2,000 | $0.033 | 3x |
| Opus | 1,000 | 2,000 | $0.165 | 15x |

**结论**：简单任务使用Haiku可节省66-93%成本

**场景2：复杂的性能优化分析（约5000 Token输出）**

| 模型 | 输入Token | 输出Token | 总成本 | 质量评分 | 性价比 |
|-----|---------|---------|--------|---------|--------|
| Haiku | 10,000 | 5,000 | $0.035 | 3/5 | 一般 |
| Sonnet | 10,000 | 5,000 | $0.105 | 4.5/5 | **最佳** |
| Opus | 10,000 | 5,000 | $0.525 | 5/5 | 低 |

**结论**：复杂任务Sonnet性价比最高，Opus仅在特别困难时使用

#### 模型选择决策树

```
开始
 │
 ├─ 任务复杂度 < 3/5？
 │   ├─ 是 → 使用 Haiku
 │   └─ 否 → 继续
 │
 ├─ 任务关键性高？
 │   ├─ 是 → 使用 Opus
 │   └─ 否 → 使用 Sonnet
 │
 └─ 预算有限？
     ├─ 是 → 使用 Haiku
     └─ 否 → 使用 Sonnet
```

## 安全注意事项

在使用Claude Code时，安全问题不容忽视。

### 不要跳过授权确认

Claude Code设置中有`Allow Dangerously Skip Permissions`选项。关闭时，Claude Code执行命令前会要求授权。

#### 为什么不能跳过

许多命令可能产生危险后果：

- `npm install`、`./gradlew bootRun`等构建命令
- 文件删除命令（`rm`、`del`）
- 数据库操作命令

**特别警告**：WSL中的危险命令（如`rm -rf /`）也会影响Windows文件系统，可能导致严重数据丢失。

#### 正确做法

Claude Code请求执行命令时：

1. **仔细阅读**命令的具体内容
2. **理解**命令会做什么操作
3. **确认**命令的影响范围
4. 只有在完全理解并确认安全后，才点击授权

> **重要**：永远不要盲目授权，也不要为了方便启用"跳过权限确认"。安全永远比便利更重要。
