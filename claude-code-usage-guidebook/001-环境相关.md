# 工作流程

## 重要说明：跨系统使用

在实际项目中，**跨系统使用是标准配置**：
- Claude Code运行在WSL/Linux环境（更好的开发工具和性能）
- IDE在Windows中使用（熟悉的界面和插件生态）

**建议掌握跨系统配置的三种方法**，具体环境配置请参考项目文档。

## 跨系统使用

典型场景：Windows上使用IDE，Claude Code运行在WSL中。

### 方法一：使用IDE远程开发功能（强烈推荐）

通过IDE的远程连接功能，直接在远程系统中开发，体验如同本地。

**VSCode示例：**
- **连接到WSL**：安装 `Remote - WSL` 扩展 → 点击左下角绿色按钮连接 → 打开项目 → 运行 `claude-code`
- **连接到远程服务器**：安装 `Remote - SSH` 扩展 → 配置SSH主机 → 连接并打开项目 → 运行 `claude-code`
- **连接到Docker容器**：安装 `Dev Containers` 扩展 → 连接到运行中的容器 → 运行 `claude-code`

**优点：**
- **最佳体验**：IDE完全运行在远程环境，无需文件同步
- **性能优秀**：避免跨文件系统访问，速度快
- **环境统一**：编辑器、终端、Claude Code都在同一系统中
- **支持插件**：可以在远程环境中安装和使用Claude Code插件
- **适用场景广**：支持WSL、SSH远程服务器、Docker容器等

**注意事项：**
- 首次连接需要在远程系统安装VSCode Server（自动完成）
- 确保远程系统已安装Claude Code CLI
- 部分扩展需要在远程环境中重新安装
- WSL模式下推荐将项目放在WSL文件系统（`~/`）而非Windows挂载目录（`/mnt/c/`）


### 方法二：直接文件系统访问

适合实时查看AI修改的场景。

**VSCode示例：**
- Windows中打开项目：`C:\projects\my-project`
- WSL终端中运行：`cd /mnt/c/projects/my-project && claude-code`
- VSCode自动检测文件变化

**优点：**
- 可以利用WSL的Linux环境优势（更好的命令行工具、包管理器等）
- 适合需要Linux环境编译/运行的项目

**注意事项：**
- 确保IDE和WSL访问的是同一个项目目录
- Windows路径 `C:\path\to\project` 对应 WSL路径 `/mnt/c/path/to/project`
- 注意文件系统权限问题
- WSL和Windows文件系统间的性能差异（建议项目放在WSL文件系统中以获得更好性能）


### 方法三：通过Git同步

适合完全隔离的环境，或希望保持清晰的代码版本管理。

**VSCode示例：**
- **WSL中开发**：运行 `claude-code` → AI完成后提交 → `git push`
- **Windows中同步**：在VSCode源代码管理面板点击"拉取" 或运行 `git pull`
- **查看修改**：通过提交历史查看AI的变更

**优点：**
- 完全隔离的开发环境，避免文件系统兼容性问题
- 清晰的版本控制，每次AI修改都有提交记录
- 适合WSL项目在独立文件系统中（如 `~/project`）的情况
- 可以利用Git分支进行实验性开发

**注意事项：**
- 需要频繁执行 git pull/push 操作
- 适合以"任务"为单位的开发模式（完成一个功能后再同步）
- 确保两边都配置了Git，且有远程仓库访问权限
- 注意 `.gitignore` 配置，避免同步不必要的文件


### 三种方法对比

| 方法 | 推荐度 | 适用场景 | 优点 | 缺点 |
|------|--------|---------|------|------|
| IDE远程开发 | ⭐⭐⭐⭐⭐ | 日常开发（推荐） | 体验最佳、性能好 | 首次配置稍复杂 |
| 直接文件访问 | ⭐⭐⭐ | 临时使用、快速上手 | 配置简单 | 跨文件系统性能差 |
| Git同步 | ⭐⭐⭐⭐ | 需要版本管理 | 清晰的提交历史 | 需要频繁同步 |

**建议：优先使用方法一（IDE远程开发）**

## 同一系统使用

### 使用IDE插件

这是**最便捷的使用方式**，适合日常开发。

**VSCode示例：**
1. 在扩展市场安装 `Claude Code` 插件
2. 打开项目文件夹
3. 点击侧边栏Claude Code图标或按 `Ctrl+Shift+P` 唤起
4. 与AI对话，文件修改会实时显示

**优点：**
- 操作体验流畅，无需切换应用
- AI能直接看到你在IDE中选中的代码
- 文件修改实时同步，所见即所得

### 使用命令行

适用于无法安装插件的情况（如受限的企业环境）。

**VSCode示例：**
1. 打开项目
2. 按 `` Ctrl+` `` 打开集成终端
3. 运行 `claude-code`
4. 在终端与AI对话，文件修改后IDE自动刷新

**优点：**
- 不依赖IDE插件，适用范围广
- AI仍然能够修改文件，只是需要手动刷新IDE查看

**注意事项：**
- 需要在终端和IDE之间切换
- 部分IDE可能需要手动刷新才能看到文件变化

## 如何选择？

| 场景 | 推荐方案 | 适用人群 |
|------|---------|---------|
| **跨系统开发（项目标准）** | **IDE远程开发** | **推荐掌握** |
| 跨系统快速上手 | 直接文件访问 | 临时使用 |
| 跨系统版本管理 | Git同步 | 需要清晰提交历史 |
| 同系统日常开发 | IDE插件方式 | 特殊场景 |
| 同系统受限环境 | 命令行方式 | 无法安装插件 |

**推荐学习路径：**
1. **第一步**：学习跨系统配置（IDE远程开发） - 参考项目环境配置文档
2. **第二步**：验证环境配置是否正确 - 参考配置检查清单
3. **第三步**：开始使用Claude Code - 参考 [2-ClaudeCode快速入门.md](2-ClaudeCode快速入门.md)

## 配置检查清单

在开始使用前，请确认以下配置：

### WSL环境检查
```bash
# 1. 检查WSL是否安装
wsl --list --verbose

# 2. 检查WSL中是否安装Claude Code CLI
wsl
claude-code --version

# 3. 检查Node.js版本（Claude Code需要）
node --version  # 应该 >= 18.0.0
```

### IDE远程连接检查（以VSCode为例）
- [ ] 已安装 `Remote - WSL` 扩展
- [ ] 左下角绿色按钮能正常连接到WSL
- [ ] 在WSL中能打开项目文件夹
- [ ] WSL终端中能运行 `claude-code` 命令

### 常见配置错误和解决方案

#### 错误1：WSL连接失败
**现象**：点击左下角绿色按钮无反应或报错

**解决方案**：
```bash
# 1. 重启WSL
wsl --shutdown
wsl

# 2. 检查WSL版本（建议使用WSL2）
wsl --set-default-version 2

# 3. 重装VSCode Server
rm -rf ~/.vscode-server
# 然后重新连接VSCode
```

#### 错误2：Claude Code命令未找到
**现象**：运行 `claude-code` 提示 command not found

**解决方案**：
```bash
# 在WSL中安装Claude Code CLI
npm install -g @anthropic/claude-code

# 或使用yarn
yarn global add @anthropic/claude-code

# 验证安装
claude-code --version
```

#### 错误3：文件权限问题
**现象**：WSL中访问Windows文件提示权限错误

**解决方案**：
```bash
# 方法1：调整文件权限
chmod -R 755 /mnt/c/projects/my-project

# 方法2：将项目移到WSL文件系统（推荐）
cp -r /mnt/c/projects/my-project ~/projects/
cd ~/projects/my-project
```

#### 错误4：路径找不到
**现象**：Windows和WSL路径不一致导致文件找不到

**路径对应关系：**
- Windows: `C:\Users\username\projects`
- WSL: `/mnt/c/Users/username/projects`
- WSL本地: `~/projects` = `/home/username/projects`

**建议**：项目统一放在WSL文件系统中（`~/projects/`），避免路径问题

### 性能优化建议
- ✅ 推荐：项目放在WSL文件系统（如 `~/projects/`）
- ❌ 避免：项目放在Windows挂载目录（如 `/mnt/c/projects/`）
- 原因：跨文件系统访问性能差约10倍
