## Skill开发问题

### Skill开发失败如何调试？

**调试步骤**：
1. **检查Skill文件格式**：确保markdown格式正确
2. **验证Prompt模板**：测试Prompt是否能独立执行
3. **检查参数传递**：确认参数名称和类型
4. **查看日志**：`~/.claude/logs/`中的错误信息

**常见错误**：
- Markdown语法错误
- Prompt中引用不存在的变量
- 文件权限问题

**相关文档**：[500-Agent与Skill开发.md](500-Agent与Skill开发.md)

---

### 如何测试Skill的可靠性？

**测试方法**：
1. **单元测试**：准备测试用例，验证输出
2. **边界测试**：测试极端情况（空输入、大文件等）
3. **真实场景**：在实际项目中试用
4. **团队Review**：让团队成员试用并反馈

**示例测试清单**：
```markdown
- [ ] 正常输入测试
- [ ] 空输入测试
- [ ] 大文件测试
- [ ] 错误处理测试
- [ ] 性能测试（Token消耗）
```

---

### Skill性能如何优化？

**优化方向**：
1. **减少文件读取**：只读必要文件
2. **精简Prompt**：去除冗余说明
3. **利用缓存**：重用前面的结果
4. **分步骤执行**：避免一次性处理

**案例**：
```markdown
优化前：读取所有文件 → 3000 Token
优化后：只读变更文件 → 500 Token
```

## Agent使用问题

### 何时使用Agent、何时直接操作？

**使用Agent的场景**：
- 独立的子任务（不依赖当前上下文）
- 需要并行处理的任务
- 探索性研究（避免污染主上下文）
- 需要隔离环境的操作

**直接操作的场景**：
- 简单快速的任务
- 需要延续当前对话上下文
- 单步骤操作
- 需要实时交互

---

### Agent性能不佳如何优化？

**优化策略**：
1. **明确任务边界**：给Agent清晰的任务描述
2. **提供必要上下文**：不要让Agent自己摸索
3. **合理设置隔离级别**：不是所有任务都需要隔离
4. **监控Agent消耗**：记录Token使用情况

## CLAUDE.md问题

### 规则太多导致上下文压力大？

**解决方案**：
1. **提炼核心规则**：只写最重要的20%
2. **使用引用**：链接到外部文档
3. **分级说明**：必须 > 推荐 > 可选
4. **定期精简**：删除过时规则

**示例**：
```markdown
## 代码规范（核心）
- 遵循团队ESLint配置
- 详细规范见：docs/style-guide.md
```

---

### 如何平衡详细度和简洁性？

**平衡原则**：
- **必须详细**：容易出错的地方
- **可以简洁**：标准做法
- **引用外部**：完整文档

**示例**：
```markdown
❌ 过度详细：
Git提交消息格式：
- 第一行不超过50字符
- 使用现在时态
- 首字母小写
... (20条规则)

✅ 恰当平衡：
Git提交：遵循Conventional Commits规范
详见：https://conventionalcommits.org
```
